# 2장. 자바스크립트란?

<br/>

## 2.1 자바스크립트의 탄생
넷스케이프 커뮤니케이션즈는 웹페이지의 보조적인 기능 수행을 위해 브라우저에서 동작하는 경량 프로그래밍언어를 개발하게 되는데,   
그것이 브랜던 아이크가 개발한 자바스크립트다.
<br/><br/>
## 2.2 자바스크립트의 표준화
마이크로소프트에서 자바스크립트 파생버전인 JScript를 출시하지만 적당히 호환되는 것이었다.   
점유율을 높이기 위해 자사브라우저에서만 동작하는 기능을 추가했지만 그 결과   
크로스 브라우징 이슈가 발생하기 시작했다.   
즉, 모든 브라우저에서 정상적으로 동작하는 웹페이지를 개발하기 어려워졌다.   
컴퓨터 시스템의 표준을 관리하는 비영리 표준화 기구 ECMA 인터내셔널에 자바스크립트 표준화를 요청한다.   
상표권 문제로 자바스크립트는 ECMAScript로 명명하며 현재까지 꾸준하게 표준 사양이 업데이트 되고 있다.   
<br/><br/>
## 2.3 자바스크립트 성장의 역사
초창기 자바스크립트는 보조적인 역할로 한정적인 용도로만 사용되었고,   
브라우저는 서버로부터 html와 css를 단순히 렌더링하는 수준이었다.   
<br/><br/>
렌더링 = html, css, javascript 문서를 브라우저에서 시각적으로 출력하는 것
<br/><br/>
### 2.3.1 Ajax
1999년 자바스크립트를 이용해 서버와 브라우저가 비동기 방식으로 통신할 수 있는 기능인   
Ajax(XMLHttpRequest 라는 이름으로)가 등장.   
이전 방식은 서버로부터 받은 html은 웹페이지가 전환되면 전체를 새로 렌더링해서 보여주는 방식.   
Ajax 등장 후엔 필요한 부분만 렌더링해서 보여주는 방식이 가능해졌다.   
<br/><br/>
### 2.3.2 jQuery
2006년 jQuery의 등장 후, 번거로웠던 DOM을 쉽게 제어할 수 있게 되었고   
크로스 브라우징 이슈도 줄어들었으며, 자바스크립트에 비해 직관적이고 배우기 쉬웠다.   
<br/><br/>
### 2.3.3 V8 자바스크립트 엔진
Ajax를 기반으로 만들어진 구글 맵스 웹앱을 통해 자바스크립트의 가능성으로 더욱 빠르게 동작하는 엔진이 필요해졌다.   
v8 자바스크립트 엔진의 등장으로 ux를 제공할 수 있는 웹앱으로 정착하게 되었으며, 프론트 영역이 주목받는 계기가 됐다.   
<br/><br/>
### 2.3.4 Node.js
2009년 v8 자바스크립트 엔진으로 빌드된 런타임 환경, node.js가 발표되었다.   
브라우저 엔진에서만 동작하던 자바스크립트를 독립시킨 것이다.   
서버 사이드 앱 개발에 주로 사용되며, 여러 장점이 있다.   
<br/><br/>
### 2.3.5 SPA 프레임워크
CBA 방법론을 기반으로 하는 SPA가 대중화 되었고, Angular, React, Vuew.js, Svelte SPA   
라이브러리/프레임워크 사용층을 확보 중이다.   
<br/><br/>
## 2.4 자바스크립트와 ECAMScript
자바스크립트는 일반적으로 뼈대를 이루는 ECMAScript와 브라우저에서 별도 지원하는   
클라이언트 사이드 Web API를 아우르는 개념이다. 자세한 사항은 MDN Web API 페이지를 참고하자.   
<br/><br/>
## 2.5 자바스크립트의 특징
자바스크립트는 html,css와 함께 웹을 구성하는 요소 중 하나로 웹 브라우저에서 동작하는 유일한 프로그래밍 언어이다.   
개발자가 별도의 컴파일 작업을 수행하지 않는 인터프리터 언어다.   
컴파일러 언어에 비해 처리속도가 느리다.   
브라우저에서 제공하는 모던 자바스크립트 엔진은 컴파일러의 장점을 결합해서 단점을 해결했다.   
프로토타입 기반의 객체지향 언어이다.   
<br/><br/>
